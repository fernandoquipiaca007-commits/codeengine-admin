## 2025-05-15 - Exposure of Supabase Service Role Key in Frontend
**Vulnerability:** The Supabase `service_role` key was prefixed with `VITE_` and used in the frontend codebase. In Vite, any environment variable starting with `VITE_` is automatically bundled into the client-side code, making it accessible to anyone via the browser's developer tools.
**Learning:** Hardcoded or environment-variable-driven administrative keys (like the Supabase service role key) bypass Row Level Security (RLS) and grant full database access. Developers often use them in admin dashboards to simplify data management, but in a client-side application, this is a critical security breach.
**Prevention:** Never prefix sensitive server-side keys with `VITE_`. Always enforce Row Level Security (RLS) on the database and use the authenticated `anon` client for all client-side data operations. If administrative actions are required, they should be performed via a secure backend proxy or serverless functions where the service role key remains private.
