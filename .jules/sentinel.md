## 2025-05-30 - Remove Supabase service role key from frontend
**Vulnerability:** The Supabase `service_role` key was exposed in the frontend through the `VITE_SUPABASE_SERVICE_ROLE_KEY` environment variable. In Vite, all `VITE_` prefixed variables are bundled into the client code. The `service_role` key bypasses Row Level Security (RLS) and provides full administrative access to the database.
**Learning:** Even if a key is named "admin" or "service role", if it's prefixed with `VITE_`, it will be exposed to the browser. Admin dashboards should still rely on authenticated user sessions and RLS for data protection rather than hardcoding privileged keys.
**Prevention:** Never prefix administrative or secret keys with `VITE_` (or `REACT_APP_` in Create React App). Use the authenticated `anon` client for frontend operations and enforce security via RLS policies on the Supabase backend. If administrative operations are necessary, they should be performed via a secure backend API that holds the `service_role` key server-side.
