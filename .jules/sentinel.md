## 2026-05-30 - Exposure of Supabase Service Role Key in Frontend
**Vulnerability:** The `VITE_SUPABASE_SERVICE_ROLE_KEY` was referenced in `src/lib/supabase-admin.ts` and included in `src/vite-env.d.ts`. In Vite, environment variables prefixed with `VITE_` are bundled into the client-side code, meaning any user could extract the service role key from the production bundle. This key bypasses Row Level Security (RLS) and grants full administrative access to the Supabase database.
**Learning:** Using the service role key in a frontend application is a critical security risk. Frontend applications should always use the `anon` key and rely on Row Level Security (RLS) and user authentication for access control. If administrative actions are needed, they should be performed via a secure backend API that holds the service role key.
**Prevention:** Never prefix sensitive secrets (like service role keys) with `VITE_` if they are not intended for the frontend. Avoid all references to administrative keys in frontend source code. Enforce RLS on all database tables and use the authenticated `anon` client for all client-side operations.
